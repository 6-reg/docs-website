[New Relic 인프라 통합에는](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 이제 지표 데이터를 New Relic으로 전송하는 AWS Security Lake 통합이 포함됩니다.

통합을 사용하여 [Security Lake](https://aws.amazon.com/security-lake/) 에서 원격 측정 데이터를 수집하고 New Relic으로 보냅니다. 서비스를 모니터링하고, 들어오는 데이터를 쿼리하고, 대시보드를 구축하여 모든 것을 한 눈에 관찰하세요.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 S3 로그 전달자를 설정하십시오. 쉽고 편리하게 응용 프로그램을 사용하는 것이 좋지만 직접 설정할 수도 있습니다.

## 구성

전용 [Amazon Security Lake 로그 전달자를](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3/) 설치하되 Lambda 트리거 대신 AWS SQS 서비스를 사용하십시오. 그에 따라 앱에서 요청하는 변수를 설정합니다.

## 로그

New Relic에서 로그를 찾으려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Logs** 로 이동하여 **Attributes** `product.name`로 설정하고 원하는 항목을 하나 또는 몇 개 선택하십시오.

현재 지원되는 로그 소스 유형은 다음과 같습니다.

* [루트 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC 흐름](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [클라우드 트레일](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [보안 허브](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)
