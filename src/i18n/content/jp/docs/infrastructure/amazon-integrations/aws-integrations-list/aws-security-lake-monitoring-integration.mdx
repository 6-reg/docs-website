[New Relic インフラストラクチャの統合に](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) は、メトリクス データを New Relic に送信する AWS Security Lake の統合が含まれるようになりました。

統合を使用して、 [Security Lake](https://aws.amazon.com/security-lake/) からテレメトリ データを収集して New Relic に送信します。サービスを監視し、受信データをクエリし、ダッシュボードを構築して、すべてを一目で確認します。

## 統合をアクティブ化する [#activate]

この統合を有効にするには、S3 ログ フォワーダーをセットアップします。簡単で便利なアプリケーションを使用することをお勧めしますが、独自に設定することもできます。

## 構成

専用の [Amazon Security Lake ログ フォワーダー](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3/) をインストールしますが、Lambda トリガーの代わりに AWS SQS サービスを使用します。それに応じて、アプリによって要求される変数を設定します。

## ログ

New Relic でログを見つけるには、 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Logs** に移動し、 **Attributes**を `product.name`に設定し、関心のあるものを 1 つまたはいくつか選択します。

現在、次のソース タイプのログがサポートされています。

* [国道53号線](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC フロー](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [クラウドトレイル](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [セキュリティハブ](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)
