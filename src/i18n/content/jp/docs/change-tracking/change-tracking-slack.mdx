Slack を使用して、変更追跡機能でフォローしているアクティビティについてチームに通知できます。たとえば、変更追跡 NerdGraph API (または以前のデプロイ REST v2 API) を使用してデプロイを記録した後、デプロイの Webhook エンドポイントを使用してチームに通知できます。

webhookの宛先は、スタックインスタンスにできます。webhookを使用して、スラックチャネルのデプロイメント通知を設定する場合は、以下の手順に従います。

1. スラックアウカントにアドミンとしてログインしてから、**アプリディレクトリ > 管理 > アプリ**の順に進みます。
2. New Relicアプリを検索してから、**設定を追加**を選択します。
3. \[チャネルに**投稿] から**、既存の Slack チャネルを選択するか、新しいチャネルを追加して、 \[**構成の追加**] を選択します。
4. オプションのリストから、webhookのURLをコピーします。
5. **[one.newrelic.com](https://one.newrelic.com/) > (ユーザーメニュー) > アカウント設定 > 統合 > 通知のデプロイ > Webhook**に移動します。
6. Slack Webhook URL を貼り付けて、変更を保存します。
7. (オプション) テスト メッセージを送信します。

Slack 通知を設定できるように変更をキャプチャする方法について質問がある場合は、 [「変更の追跡を開始する](/docs/change-tracking/change-tracking-introduction#start-tracking)」を参照してください。
