You can think of our synthetic monitors as crash test dummies for your websites, applications, and API endpoints. When you deploy a synthetic monitor, an API check or Chrome browser runs through your most important workflows, testing your critical endpoints and reporting the results to New Relic. By mimicking the behaviors of customers, your monitors test how well your web apps handle traffic. 

The below tutorial walks you through creating your first ping, scripted browser, and scripted API monitors. When you finish, you'll be ready to deploy your first batch of synthetic monitors in New Relic—all without installation.

## Create a scripted browser monitor [#browser]

Ping monitors won't execute JavaScript because they aren't a full browser, just an HTTP client. Simple browser monitors, while using a Google Chrome instance, don't perform complex checks on your web page. 

Scripted browser monitors give you full Javascript functionality so your monitors can check the availability of the individual elements in your web app. 

<SideBySide>
    <Side>
    ### Go to a URL [#advanced-url]
    The beginning of all scripts specifies the URL the monitor should check.

    </Side>
    <Side>

```js
//Visit ELEPHANT
$webDriver.get("ELEPHANT");  
```

</Side>
</SideBySide> 

<SideBySide>
    <Side>
    ### Interact with a page element [#advanced-nav]

Once your monitor goes to a URL, it scans for then interacts with the page's available elements. This script: 
    1. Finds the **About** page via link text, then clicks the link: `$webDriver.findElement($selenium.By.linkText("About")).click()` 
    2. Searches for the partial string `Home` to find the the **Example site Home** buton, then clicks the link: `$webDriver.findElement($selenium.By.partialLinkText("Home")).click();});`
    </Side>
    <Side>

```js
$webDriver.get("http://example.com/").then(function(){
//Find a link with display text that matches `About`, then click that link.
    return $webDriver.findElement($selenium.By.linkText("About")).click();
}).then(function(){
    return $webDriver.findElement($selenium.By.partialLinkText("Home")).click();
});
```
</Side>
</SideBySide> 

<SideBySide>
    <Side>
### Search a website [#advanced-search]

Beyond text sweeps, your scripted browser monitor can locate elements via their `class`, `id`, `name`, or XPath. This script:

1. Goes to your site.
2. Locates a search box via its XPath, then types `relic`.
3. Locates a submit button via its XPath, then clicks it to submit the search.

    </Side>
    <Side>
```js
$webDriver.get("http://example.com/static/companyBlog.jsp").then(function(){
//Find the search field by specifying its `id`, then enter `relic`.
    return $webDriver.findElement($selenium.By.xpath("//h4[text()='Blog Search']/following-sibling::div/input")).sendKeys("relic");
}).then(function(){
//Click the search button.
    return $webDriver.findElement($selenium.By.xpath("//h4[text()='Blog Search']/following-sibling::div//button")).click();
});
```
    </Side>
</SideBySide>

<SideBySide>
    <Side>
##  Wait for a page element [#wait-for-element] 
    
    Your script may need to wait an interval before reporting a failure, since images or dynamic elements can take time to generate on a page. This script:

1. Goes to the example site.
2. Finds the **Support** dropdown via `HTML ID`, then clicks it.
3. Waits up to 20 seconds for the **FAQ** button to appear, then clicks it.
    </Side>
    <Side>
```js
$webDriver.get("http://example.com/").then(function(){
    return $webDriver.findElement($selenium.By.id("supportDropDown")).click();
}).then(function(){
    return $webDriver.waitForAndFindElement($selenium.By.id("supportFAQLink"), 20000).then(function(aboutPage){
        return aboutPage.click();
    })
});
```  </Side>
</SideBySide>

<SideBySide>
    <Side>
## Log in to a website [#advanced-login]

You can take your stored secure credentials to test certain page elements on your web apps. For scripting purposes, we recommend that you create new credentials unique to your script. Don't use your personal credentials or reuse credentials when running a monitor. This script:

1. Loads `http://example.com/login.jsp`
2. Locates the username field by the element name, then submit a secure username [via our secure credentials feature.](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
3. Locates the password field by the element name, then submit a secure password.
4. Finds the login button via its XPath, then clicks to submit the account information.

For more information about using secure credentials, see [Store secure credentials](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers).
    </Side>
    <Side>
```js
$webDriver.get("http://example.com/login.jsp").then(function(){
    return $webDriver.findElement($selenium.By.name("username")).sendKeys($secure.SECURE_USERNAME);
}).then(function(){
//Find the password field by specifying its name, then submits a secured password.
    return $webDriver.findElement($selenium.By.name("password")).sendKeys($secure.SECURE_PASSWORD);
}).then(function(){
//Find and click the login button.
    return $webDriver.findElement($selenium.By.xpath("//input[@value='Login']")).click();
});
```
  </Side>
